实验3 运用云平台搭建个人博客

实验目的

熟悉个人博客的搭建流程，运用云平台搭建个人博客，并将实验报告上传。

具体包括：注册Github账号，搭建Hexo环境并实现个人博客搭建，撰写实验报告，并将实验报告上传至自己的博客。

实验步骤与内容

1、环境搭建

Hexo 基于 Node.js，搭建过程中还需要使用 npm（Node.js 已带） 和 git，因此先搭建本地操作环境，安装 Node.js 和 Git。在其他课程的实验中我曾经安装过对应组件。此处仅展示其版本号

2、连接Github

此前已经注册过Github账号，因此直接进行设置。

右键 -> Git Bash Here ，设置用户名和邮箱。

之后创建SSH密钥，因网络原因，我重复做了多次实验，此处仅展示其中某一次（非最终）的创建结果。

之后添加密钥，进入 [C:\Users\用户名.ssh] 目录（要勾选显示“隐藏的项目”），用记事本打开公钥 id_rsa.pub 文件并复制里面的内容。对应的目录下文件结构见下图

登陆 GitHub ，进入 Settings 页面，选择左边栏的 SSH and GPG keys，点击 New SSH key。取个标题，粘贴复制的 id_rsa.pub 内容到 Key 中，点击 Add SSH key 完成添加。添加过程和完成后的结果见下图。

之后验证连接，打开Git Bash，输入对应指令，显示“Hi xxx! You’ve successfully……” 即连接成功。因网络原因，实验进行多次，此处仅给出某一次成功连接的示意图。

7、更换主题

为了使网站更有辨识度，我选择更换一个主题。在Hexo官方网站（https://hexo.io/themes ）上有很多主题，在其中我使用了Matery主题，其具有以下优点：

  • 简单漂亮，文章内容美观易读

  • Material Design 设计

  • 响应式设计，博客在桌面端、平板、手机等设备上均能很好的展现

  • 首页轮播文章及每天动态切换 Banner 图片

  • 瀑布流式的博客文章列表（文章无特色图片时会有 24 张漂亮的图片代替）

  • 时间轴式的归档页

  • ……

其多元化的功能将在以后逐渐实现，在此我仅进行主题的更换。

首先下载主题，可通过直接访问https://blinkfox.github.io/ ，或者在themes文件夹下使用git clone命令来下载：

结论分析与体会：

在本次实验中，我学习使用了GitHub Pages来搭建 Hexo 静态博客网站，通过ssh密钥将Hexo和GitHub进行连接，最终成功搭建了个人网站。虽然因为网络原因，总是遇上上传失败等问题，或者其他的问题导致需要重新开始实验，但是在多次实验的过程中，我对此也有了更加深刻的理解，也算是因祸得福。

下面给出hexo的常用命令

指令 用途
hexo new “name” 新建文章
hexo new page “name” 新建页面
hexo g 生成页面
hexo d 部署
hexo g -d 生成页面并部署
hexo s 本地预览
hexo clean 清除缓存和已生成的静态文件
hexo help 帮助

另外我还了解了一些hexo的相关知识，此处给出：

Hexo是一个基于Node.js的静态博客框架，可以快速、简洁且高效地生成静态网页。Hexo支持Markdown和大多数Octopress插件，有强大的API和无限的扩展性。Hexo还可以一键部署到GitHub Pages, Heroku或其他平台。

Hexo的优点有：

  • 生成速度快，性能高

  • 主题丰富，自定义灵活

  • 文档完善，社区活跃

  • 兼容多种模板引擎和NPM包

Hexo的缺点有：

  • 需要一定的Node.js基础和命令行操作

  • 需要手动备份数据和配置文件

  • 需要定期更新依赖包和插件
